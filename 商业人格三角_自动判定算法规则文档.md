# 商业人格三角 · 自动判定算法规则文档

**版本**：v1.0  
**适用**：MVP 人格画像模块（`shareCard.creatorType`）  
**输入**：动态权重调整后的六维度得分 + 用户画像 + 计划书文本特征  
**输出**：`creatorCode`（如 `B-HL`）+ `creatorType`（如 `饼王`）

---

## 1. 判定流程总览

```
┌─────────────────────────────────────────┐
│  输入：六维度得分（0-100）+ 用户画像        │
│         + 计划书文本特征（可选）            │
├─────────────────────────────────────────┤
│  Step 1: 计算价值锚点得分 → 确定 B/H/S    │
│  Step 2: 计算决策温度得分 → 确定 H/C      │
│  Step 3: 计算时间透镜得分 → 确定 S/L      │
├─────────────────────────────────────────┤
│  输出：creatorCode（如 B-HL）            │
│         creatorType（如 饼王）            │
│         spiritAnimal / nemesis / pair     │
└─────────────────────────────────────────┘
```

**判定时机**：在「动态权重评分」完成后、「报告生成」之前。使用**加权后的最终六维度得分**作为输入。

---

## 2. Step 1：价值锚点判定（Builder / Hunter / Storyteller）

### 2.1 核心公式

对六维度得分计算三类锚点的**加权特征分**：

```python
# 输入：六维度最终得分（0-100）
dimensions = {
    "marketSize": 82,
    "painPoint": 74,
    "businessModel": 66,
    "competitiveAdvantage": 71,
    "executionFeasibility": 62,
    "viralityNarrative": 88
}

# 造局者得分（Builder）
B_score = (
    dimensions["painPoint"] * 0.30 +
    dimensions["competitiveAdvantage"] * 0.30 +
    dimensions["executionFeasibility"] * 0.25 +
    dimensions["marketSize"] * 0.10 +
    dimensions["viralityNarrative"] * 0.05
)

# 捕猎者得分（Hunter）
H_score = (
    dimensions["businessModel"] * 0.35 +
    dimensions["marketSize"] * 0.30 +
    dimensions["executionFeasibility"] * 0.20 +
    dimensions["competitiveAdvantage"] * 0.10 +
    dimensions["viralityNarrative"] * 0.05
)

# 叙事者得分（Storyteller）
S_score = (
    dimensions["viralityNarrative"] * 0.40 +
    dimensions["marketSize"] * 0.25 +
    dimensions["painPoint"] * 0.20 +
    dimensions["businessModel"] * 0.10 +
    dimensions["executionFeasibility"] * 0.05
)
```

### 2.2 判定规则

| 条件 | 结果 | 说明 |
|------|------|------|
| `B_score` 为三者最高，且领先第二名 ≥ 5 分 | **B（造局者）** | 核心驱动力是「解决问题 + 构建壁垒」 |
| `H_score` 为三者最高，且领先第二名 ≥ 5 分 | **H（捕猎者）** | 核心驱动力是「商业模式 + 市场套利」 |
| `S_score` 为三者最高，且领先第二名 ≥ 5 分 | **S（叙事者）** | 核心驱动力是「传播叙事 + 认知溢价」 |
| 最高分与第二名差距 < 5 分 | 进入 **2.3 混合判定** | 需要用户画像辅助裁决 |

### 2.3 混合判定（差距 < 5 分）

当两类得分接近时，按以下优先级裁决：

```python
# 假设 B_score 与 S_score 接近（差距 < 5）
if userProfile["goal"] in ["competition", "social_media"]:
    result = "S"  # 比赛/社媒目的，叙事优先
elif userProfile["identity"] == "creator":
    result = "S"  # 创作者身份，叙事优先
elif userProfile["identity"] in ["founder", "employee"]:
    result = "B"  # 创始人/员工，造局优先
elif userProfile["riskPreference"] == "aggressive" and dimensions["marketSize"] > 80:
    result = "H"  # 激进 + 大市场，捕猎优先
else:
    result = 三者中的最高分  # 默认取数值最高
```

**常见混合场景裁决表**：

| 接近组合 | 裁决规则 | 典型场景 |
|----------|----------|----------|
| **B vs H** | `stage=launched` → H；`stage=idea` → B；`businessModel > painPoint` → H | 有产品但商业模式强 vs 有痛点但产品早期 |
| **B vs S** | `goal=competition/social_media` → S；`executionFeasibility > 75` → B；否则 S | 创意项目 vs 硬核项目 |
| **H vs S** | `identity=creator` → S；`businessModel > 80` → H；否则 S | 内容创业 vs 贸易套利 |

## 5. 边界情况处理

### 5.1 六维度缺失字段的影响

如果计划书缺失关键字段（MVP Spec 中的 `missingFields`），人格判定需要调整：

| 缺失字段 | 对人格判定的影响 |
|----------|----------------|
| `businessModel` | `H_score` 上限锁死在 60；`B_score` 中 `executionFeasibility` 权重临时 +0.10 |
| `competitors` | `B_score` 中 `competitiveAdvantage` 权重归零，改用 `marketSize * 0.15` 替补 |
| `problem` | `B_score` 中 `painPoint` 权重归零，人格偏向 H 或 S |
| `solution` | `B_score` 整体 -10 分 |
| `targetUsers` | `S_score` 中 `viralityNarrative` 权重 -0.10 |

**兜底规则**：如果缺失字段 ≥ 3 个，系统提示「计划书信息不足，人格判定可能偏差」，并默认输出 **缝合怪 (B-HS)** 或 **快钱游侠 (H-HS)**（根据 `businessModel` 是否缺失决定）。

### 5.2 极端分数处理

| 极端情况 | 处理规则 |
|----------|----------|
| 某维度 = 0（完全未提及） | 不参与该维度的优势判定，但参与劣势判定（如 `businessModel=0` 直接触发 `long_bonus + 15`） |
| 六维度全 > 80（六边形战士） | 优先判定为 **B-CL（基建狂魔）** 或 **S-CL（重新定义大师）**，具体看 `competitiveAdvantage` 与 `viralityNarrative` 谁更高 |
| 六维度全 < 40（全拉胯） | 输出 **Chaos Operator（混乱运营者）** 彩蛋人格，代码为 `X-HX`，不进入 12 型体系，作为吐槽彩蛋 |

### 5.3 与投资人权重的交叉验证

Trump 和 Musk 的权重体系会影响六维度最终得分，因此**同一份计划书在不同投资人视角下，人格可能不同**。这是预期行为。

| 投资人 | 人格偏移倾向 |
|--------|-------------|
| **Trump** | 提高 `businessModel` 和 `marketSize` 权重 → 更容易出 **H 族（捕猎者）**；`executionFeasibility` 权重中等 → B 族减少 |
| **Musk** | 提高 `marketSize` 和 `viralityNarrative` 权重 → **S 族（叙事者）** 和 **B-HL（饼王）** 增加；`businessModel` 权重低 → H 族减少 |

**实现方式**：人格判定函数接收 `investorPersona` 参数，在动态权重调整后分别计算两次（如果用户选择对比模式）。

## 6. 完整伪代码实现

```python
def determine_creator_type(dimensions, user_profile, missing_fields, investor_persona, text=""):
    """
    dimensions: dict, 六维度最终得分 (0-100)
    user_profile: dict, {identity, stage, goal, riskPreference}
    missing_fields: list, 缺失字段
    investor_persona: str, "trump" | "musk"
    text: str, 计划书原始文本（可选）
    """

    # ========== Step 1: 价值锚点 ==========
    # 缺失字段修正
    b_weights = {"painPoint": 0.30, "competitiveAdvantage": 0.30, 
                 "executionFeasibility": 0.25, "marketSize": 0.10, "viralityNarrative": 0.05}
    h_weights = {"businessModel": 0.35, "marketSize": 0.30, 
                 "executionFeasibility": 0.20, "competitiveAdvantage": 0.10, "viralityNarrative": 0.05}
    s_weights = {"viralityNarrative": 0.40, "marketSize": 0.25, 
                 "painPoint": 0.20, "businessModel": 0.10, "executionFeasibility": 0.05}

    # 应用缺失字段惩罚
    if "competitors" in missing_fields:
        b_weights["competitiveAdvantage"] = 0
        b_weights["marketSize"] += 0.15
    if "businessModel" in missing_fields:
        h_weights["businessModel"] = 0
        h_weights["executionFeasibility"] += 0.15

    B_score = sum(dimensions[k] * v for k, v in b_weights.items())
    H_score = sum(dimensions[k] * v for k, v in h_weights.items())
    S_score = sum(dimensions[k] * v for k, v in s_weights.items())

    # 判定价值锚点
    scores = {"B": B_score, "H": H_score, "S": S_score}
    sorted_scores = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    anchor = sorted_scores[0][0]

    # 混合判定（差距 < 5）
    if sorted_scores[0][1] - sorted_scores[1][1] < 5:
        anchor = resolve_anchor_tie(sorted_scores, user_profile, dimensions)

    # ========== Step 2: 决策温度 ==========
    hot = 0
    cold = 0

    # 基础分
    rp = user_profile["riskPreference"]
    if rp == "aggressive": hot += 25; cold += 5
    elif rp == "conservative": hot += 5; cold += 25
    else: hot += 15; cold += 15

    # 维度修正
    if dimensions["viralityNarrative"] > 85: hot += 10
    if dimensions["painPoint"] > 80 and dimensions["businessModel"] < 55: hot += 12
    if dimensions["marketSize"] > 85 and dimensions["executionFeasibility"] < 60: hot += 8
    if user_profile["identity"] == "student" and user_profile["goal"] == "competition": hot += 10

    if dimensions["executionFeasibility"] > 80 and dimensions["businessModel"] > 70: cold += 12
    if dimensions["competitiveAdvantage"] > 80 and dimensions["marketSize"] < 70: cold += 8
    if user_profile["identity"] == "employee" and user_profile["goal"] == "startup_validation": cold += 10
    if rp == "conservative" and user_profile["stage"] == "launched": cold += 8

    # 文本修正（可选）
    hot += min(sum(1 for w in ["风口","快速","爆发","颠覆","梦想"] if w in text) * 3, 15)
    cold += min(sum(1 for w in ["数据","验证","模型","ROI","精益"] if w in text) * 3, 15)

    if hot > cold + 5: temp = "H"
    elif cold > hot + 5: temp = "C"
    else:
        if user_profile["stage"] == "idea": temp = "H"
        elif user_profile["stage"] == "launched": temp = "C"
        else: temp = "H" if user_profile["goal"] in ["competition","social_media"] else "C"

    # ========== Step 3: 时间透镜 ==========
    short = 0
    long = 0

    # 阶段
    stage_map = {"idea": (20,5), "prototype": (12,12), "launched": (5,20)}
    s, l = stage_map[user_profile["stage"]]
    short += s; long += l

    # 目的
    goal_map = {
        "fundraising": (5,20), "startup_validation": (8,15),
        "competition": (18,7), "coursework": (15,8), "social_media": (22,3)
    }
    s, l = goal_map[user_profile["goal"]]
    short += s; long += l

    # 维度修正
    if dimensions["businessModel"] > 75 and dimensions["executionFeasibility"] > 70: short += 10
    if anchor == "H" and dimensions["businessModel"] > 70: short += 8
    if user_profile["identity"] in ["student", "small_business_owner"]: short += 8

    if dimensions["marketSize"] > 85 and dimensions["businessModel"] < 60: long += 12
    if anchor == "B" and dimensions["competitiveAdvantage"] > 75: long += 10
    if user_profile["identity"] == "founder" and user_profile["goal"] == "fundraising": long += 10
    if anchor == "S" and dimensions["viralityNarrative"] > 80: long += 8

    # 文本修正（可选）
    short += min(sum(1 for w in ["MVP","验证","试错","现金流","窗口期"] if w in text) * 3, 15)
    long += min(sum(1 for w in ["长期","护城河","复利","基础设施","终局"] if w in text) * 3, 15)

    if short > long + 5: lens = "S"
    elif long > short + 5: lens = "L"
    else:
        if rp == "aggressive": lens = "L"
        elif rp == "conservative": lens = "S"
        else: lens = "L" if anchor == "B" else "S"

    # ========== 组装结果 ==========
    code = f"{anchor}-{temp}{lens}"

    # 映射表（12型）
    type_map = {
        "B-HS": "缝合怪", "B-HL": "饼王", "B-CS": "表哥", "B-CL": "基建狂魔",
        "H-HS": "快钱游侠", "H-HL": "时间刺客", "H-CS": "黄牛", "H-CL": "老阴逼",
        "S-HS": "操盘手", "S-HL": "邪教头子", "S-CS": "品牌法医", "S-CL": "重新定义大师"
    }

    # 极端情况：六边形战士
    if all(v > 80 for v in dimensions.values()):
        code = "B-CL" if dimensions["competitiveAdvantage"] > dimensions["viralityNarrative"] else "S-CL"

    # 极端情况：全拉胯彩蛋
    if all(v < 40 for v in dimensions.values()):
        code = "X-HX"  # 彩蛋：混沌运营者
        return {
            "creatorCode": "X-HX",
            "creatorType": "混沌运营者",
            "creatorClan": "待定",
            "isEasterEgg": True
        }

    return {
        "creatorCode": code,
        "creatorType": type_map.get(code, "未知人格"),
        "creatorClan": {"B": "造局者", "H": "捕猎者", "S": "叙事者"}[anchor],
        "spiritAnimal": get_spirit_animal(code),
        "nemesis": get_nemesis(code),
        "goldenPair": get_golden_pair(code),
        "scores": {"B": round(B_score), "H": round(H_score), "S": round(S_score),
                   "Hot": hot, "Cold": cold, "Short": short, "Long": long}
    }


def resolve_anchor_tie(sorted_scores, user_profile, dimensions):
    """混合人格平局裁决"""
    first, second = sorted_scores[0][0], sorted_scores[1][0]

    # B vs S
    if set([first, second]) == {"B", "S"}:
        if user_profile["goal"] in ["competition", "social_media"]: return "S"
        if user_profile["identity"] == "creator": return "S"
        if user_profile["identity"] in ["founder", "employee"]: return "B"
        return first  # 默认取最高

    # B vs H
    if set([first, second]) == {"B", "H"}:
        if user_profile["stage"] == "launched": return "H"
        if user_profile["stage"] == "idea": return "B"
        if dimensions["businessModel"] > dimensions["painPoint"]: return "H"
        return first

    # H vs S
    if set([first, second]) == {"H", "S"}:
        if user_profile["identity"] == "creator": return "S"
        if dimensions["businessModel"] > 80: return "H"
        return "S"

    return first
```

## 7. 判定示例

### 示例 A：学生比赛项目（高叙事）

**输入**：
```json
{
  "dimensions": {"marketSize": 70, "painPoint": 65, "businessModel": 45, 
                 "competitiveAdvantage": 60, "executionFeasibility": 50, "viralityNarrative": 92},
  "userProfile": {"identity": "student", "stage": "idea", "goal": "competition", "riskPreference": "aggressive"},
  "missingFields": ["businessModel"]
}
```

**计算过程**：
- Step 1: `S_score` = 92*0.40 + 70*0.25 + 65*0.20 + 45*0.10 + 50*0.05 = **78.5**；`B_score` = **62.1**；`H_score` = **52.3** → **S（叙事者）**
- Step 2: `base_hot` = 25, `hot_bonus` = 10(叙事>85) + 10(学生+比赛) = **45**；`cold` = 5 → **H（热血）**
- Step 3: `base_short` = 20(idea) + 18(competition) = 38；`short_bonus` = 8(学生)；`long_bonus` = 8(S+叙事>80) → **Short=46, Long=13** → **S（短焦）**

**结果**：`S-HS` → **操盘手**

### 示例 B：创始人融资项目（高市场 + 高壁垒）

**输入**：
```json
{
  "dimensions": {"marketSize": 95, "painPoint": 70, "businessModel": 55, 
                 "competitiveAdvantage": 85, "executionFeasibility": 60, "viralityNarrative": 75},
  "userProfile": {"identity": "founder", "stage": "prototype", "goal": "fundraising", "riskPreference": "balanced"}
}
```

**计算过程**：
- Step 1: `B_score` = 70*0.30 + 85*0.30 + 60*0.25 + 95*0.10 + 75*0.05 = **74.75**；`H_score` = **67.25**；`S_score` = **71.5** → **B（造局者）**（与 S 差距 3.25 < 5，但 founder+高壁垒 → B）
- Step 2: `base_hot` = 15, `base_cold` = 15；`cold_bonus` = 0；`hot_bonus` = 0（marketSize>85但execution=60，不满足；叙事75不满足）→ 平局；`stage=prototype`, `goal=fundraising` → **C（冷血）**
- Step 3: `base_short` = 12(prototype) + 5(fundraising) = 17；`base_long` = 12 + 20 = 32；`long_bonus` = 12(市场>85+商业模式<60) + 10(B+竞争>75) + 10(创始人+融资) = **32**；`short_bonus` = 0 → **Long=64, Short=17** → **L（长焦）**

**结果**：`B-CL` → **基建狂魔**
